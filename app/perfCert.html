Documented Performance Exceptions
Mobile Accordion — height: 0 → auto
The mobile accordion uses height animation rather than the preferred scaleY approach. scaleY was evaluated but produces incorrect visual layout — the accordion content renders at its full height visually but the surrounding flow does not adjust, causing elements below to overlap.

Trigger: Single user tap — not a continuous or scroll-driven animation.
Frequency: Once per accordion open/close interaction.
Layout events: ~14 per cycle across a 400ms duration.
Impact: Imperceptible on modern mobile hardware at this frequency.
Accepted — discrete tap, layout correctness required
Mobile Burger Icon — position: relative → absolute
The hamburger-to-X animation uses a position property change alongside rotation to achieve the two-line cross effect. Transform-only alternatives (translateY to overlap lines then rotate) were evaluated but did not produce a satisfactory visual result within the existing button container constraints.

Trigger: Single user tap — not a continuous animation.
Frequency: Once per menu open/close interaction.
Layout events: 1–2 per tap at animation start.
Impact: Single layout event, contained to the button element.
Accepted — discrete tap, visual correctness required
GPU Optimisation Techniques
✦
will-change: transform
Applied to all animated elements — promotes to compositor layer before animation fires
✦
gsap.killTweensOf()
Called before every tween — prevents conflicting animations on rapid hover
✦
overwrite: "auto"
Surgical overwrite — only conflicting properties are cancelled, not entire tweens
✦
clipPath animation
Image curtain reveal uses clipPath — runs on compositor thread, zero layout impact
✦
pointer-events: none default
Hidden dropdowns block no events — no invisible hit areas interfering with scroll
✦
Image stack pre-rendered
All images rendered at mount — no DOM insertion during interaction, zero reflow
✦
useRef for state tracking
openPanelIndex, activeImgIndex, activeLinkIndex stored in refs — zero re-renders
✦
requestAnimationFrame via GSAP
All tweens scheduled through GSAP's rAF — batched, never blocking the main thread
